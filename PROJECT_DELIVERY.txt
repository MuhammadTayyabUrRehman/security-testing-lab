===============================================================================
VULNERABILITY DETAILS
===============================================================================

VULNERABILITY #1: BROKEN ACCESS CONTROL (CRITICAL)
─────────────────────────────────────────────────
Location: /profile endpoint
Severity: CRITICAL
OWASP: A01:2021

Issue:
    Any logged-in user can view any other user's profile
    No authorization checks verify user ownership
    User IDs are sequential and enumerable

Impact:
    Data breach - Access to private user information
    User enumeration - Can discover all system users
    Privacy violation - Unauthorized data disclosure

Test:
    1. Login as 'user'
    2. Visit: http://127.0.0.1:5000/profile?id=1
    3. Result: Can see admin's profile information

Code Location: app.py, /profile route (~line 125)


VULNERABILITY #2: REFLECTED XSS (HIGH)
──────────────────────────────────────
Location: /search endpoint, query parameter 'q'
Severity: HIGH
OWASP: A03:2021 - Injection

Issue:
    User input directly reflected in HTML without sanitization
    No HTML escaping applied
    Scripts can execute in browser context

Impact:
    Session hijacking through cookie theft
    Malware distribution
    Credential harvesting
    Phishing attacks

Test:
    Visit: http://127.0.0.1:5000/search?q=<script>alert('XSS')</script>
    Result: Alert box appears

Code Location: app.py, /search route (~line 100)


VULNERABILITY #3: WEAK AUTHENTICATION (HIGH)
──────────────────────────────────────────────
Location: /login endpoint
Severity: HIGH
OWASP: A07:2021 - Identification and Authentication Failures

Issues:
    - Plaintext passwords in code
    - No account lockout mechanism
    - No rate limiting
    - No MFA
    - Predictable credentials

Impact:
    Brute force attacks possible
    Unauthorized account access
    Weak password protection

Test:
    Try 100 failed login attempts
    Result: No lockout, unlimited attempts possible

Code Location: app.py, USERS dictionary + /login route


VULNERABILITY #4: CRYPTOGRAPHIC FAILURES (CRITICAL)
─────────────────────────────────────────────────────
Location: USERS dictionary in app.py
Severity: CRITICAL
OWASP: A02:2021

Issue:
    Passwords stored in plaintext
    No hashing or encryption
    Visible in source code

Impact:
    If code is compromised, all credentials exposed
    No protection against data breach
    Violates all security standards

Test:
    Open app.py, search for "USERS = {"
    Result: Passwords visible in plain text

Code Location: app.py, line ~50


VULNERABILITY #5: SECURITY MISCONFIGURATION (HIGH)
───────────────────────────────────────────────────
Location: app.secret_key
Severity: HIGH
OWASP: A05:2021 - Security Misconfiguration

Issue:
    Session secret key is weak and simple
    Easy to guess: 'weak_secret_key_for_lab'
    Could allow session token forgery

Impact:
    Attacker can forge valid session tokens
    Potential authentication bypass
    Session hijacking

Test:
    Open app.py, search for "app.secret_key"
    Result: Simple, guessable key string

Code Location: app.py, line ~15
